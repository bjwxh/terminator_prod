# Changelog: 2026-03-26

## 🛡️ Core Engine: "Effective Position" Subtraction Logic
*   **Intelligent Reconciliation**: Overhauled the trade execution logic to calculate the **Effective Account Position** (Sum of filled contracts + working orders) before submitting new chunks.
*   **Partial Overlap Aggregation**: The app now subtracts manual working contracts from strategy target quantities leg-by-leg. If you confirm a 10-lot trade but have 4 lots manually working, the app only submits 6.
*   **Auto-Dismissal of Redundancy**: The trade confirmation modal now automatically closes if it detects the trader has manually neutralized the discrepancy at the broker.
*   **Execution Safety**: Verified that the $0.10 price offset is applied to every individual order chunk, ensuring consistent pricing during multi-leg execution.

## 📊 Dashboard UI Improvements
*   **Active Configuration Fix**: Refactored the 'Active Configuration' table rendering to ensure server-side settings (thresholds, timings) always populate on every frontend connect.
*   **Sound & Visual Alerts**: Restored audio chimes for trade signals and verified email alerts for real-time trade requests.

## 📈 EOD Reporting & Analytics (Production VM)
*   **EOD Automation**: Configured a `cron` job on the production VM to auto-run the End-of-Day report every weekday at **3:05 PM CDT**.
*   **Data Integration**: Synchronized historical trading data from MacBook storage to the production VM to maintain a unified performance ledger.
*   **Headless Report Generation**: Updated the analytics engine to use the `Agg` Matplotlib backend for chart generation on the headless server.
*   **Stability Fix**: Resolved a critical **KeyError** where report generation failed due to a mismatch in timestamp keys (`ts` vs `timestamp`).

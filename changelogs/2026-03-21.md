# Changelog — 2026-03-21

Added real-time performance tracking with interactive charts for the SPX index and portfolio Net PnL (Live vs. Sim).

---

## New Features

### 📈 Historical Replay on Startup
**File:** `server/core/monitor.py`

The Trading Server now re-simulates the current day's performance from the database when it starts (from 8:30 AM to current time). This allows the dashboard to "replay" the day's movement immediately upon connection, even after a server restart.

**Implementation (Antigravity):**
- Updated `_run_historical_simulation` to support a `collect_history` mode.
- Populates a new `self.session_history` array on startup with timestamps, SPX prices, and PnL snapshots at 30-second intervals.

---

### 📊 Interactive Dashboard Charts
**Files:** `server/static/index.html`, `server/static/style.css`, `server/static/app.js`

Integrated **Chart.js** into the "Session Stats" tab to provide visual tracking of index price and trading performance.

**Implementation (Antigravity):**
- **Dual Chart Layout**: Added SPX Price Tracking (left) and Net PnL Comparison (right).
- **PnL Overlay**: The Performance chart overlays **Live PnLs** (Solid Green) and **Sim PnLs** (Dashed Blue) for real-time drift analysis.
- **Dynamic Scaling**: The PnL chart Y-axis automatically scales based on the day's range.
- **Responsive Design**: Implemented CSS Flexbox logic to show charts side-by-side on desktop and stacked vertically on mobile devices.

---

### 🛰️ WebSocket History Sync
**File:** `server/api/ws.py`

Modified the WebSocket protocol to ensure clients receive full historical data immediately upon connection.

**Implementation (Antigravity):**
- Added a `history_init` message type that pushes the server's `session_history` buffer to new clients.
- Included the latest estimated `spx` price in every 500ms `state_update` broadcast.

---

## Core Improvements

### Real-Time Performance Monitor
**File:** `server/core/monitor.py`

The main monitoring loop now records a data point every 30 seconds into the session buffer. This ensures that even if no trades occur, the chart continues to update with the latest price and mark-to-market PnL.

---

## Bug Fixes & Refinements
- **SPX Estimation Cache**: Added `_last_spx_price` to the `LiveTradingMonitor` to prevent "flickering" 0.0 values on the dashboard if a single option chain snapshot is partially incomplete.
- **Dependency Management**: Integrated Chart.js and `date-fns` time adapter via CDN to keep the application lightweight and avoid server-side build dependencies.

# Changelog: 2026-03-27

## ⚡ Schwab API Optimization: "Lean Sync" (2-Call Pattern)
*   **API Load Reduction**: Successfully refactored the broker synchronization logic from a 4-call pattern to a **consolidated 2-call pattern** (`get_account` + `get_orders_for_account`).
*   **Increased Performance**: Established a stable 5-second polling cadence for all account data (Balance, Positions, Trades, Working Orders), providing a 60% safety buffer under Schwab's 60 calls/min rate limit.
*   **Time-Window Filtering**: Optimized order retrieval to focus exclusively on the current trading day starting from **08:30 AM Chicago time**, significantly reducing JSON processing overhead.
*   **Heartbeat Consolidation**: Eliminated the redundant `run_broker_heartbeat` background task. The connection state is now monitored directly within the data sync loop, improving system simplicity and reliability.

## 🛡️ Complex Strategy Support (Iron Condors)
*   **Deep Multi-Leg Awareness**: Implemented the `_convert_order_to_trade` helper to correctly process strategy-based orders. The system now treats a 4-leg Iron Condor as a single unified trade record rather than disconnected individual fills.
*   **Broker Sync Reliability**: Verified that fresh broker data correctly clears the `awaiting_broker_sync` flag, ensuring the UI accurately reflects live activity following manual interventions.

## 🛠️ Diagnostic & Testing Tools
*   **Order Audit Tool**: Developed a high-precision testing script `tmp/test_get_account.py` to troubleshoot broker-side order invisibility. 
*   **Chicago Time Alignment**: Synchronized all polling activities with the `America/Chicago` timezone to ensure perfect alignment with market open/close activity and scheduled EOD reporting.
*   **Logging Refinement**: Set the "Fast Model Update" simulation messages to `DEBUG` level to prioritize live broker activity in the production logs.

## 📈 System Health & Stability
*   **Automatic Re-Auth**: Integrated connection-failure detection into the sync loop; the monitor now automatically re-initializes the Schwab client after 3 consecutive failures to recover from session timeouts or network drops.

## 🔄 Soft Bootstrap Overhaul (Sim vs. Live Sync)
*   **Intelligent First-IC Matching**: Refactored the historical catch-up logic to identify and "seed" the initial Iron Condor entry for each sub-strategy by matching live fills within a **1-hour window** of the strategy's scheduled start time.
*   **Mutual Clarity Rule**: Implemented conflict resolution for ambiguous trades; if a live trade matches multiple strategies or a strategy sees multiple live ICs in its window, the app gracefully falls back to simulation logic to avoid incorrect state initialization.
*   **Simulation Management Dominance**: Once a sub-strategy has its initial position (matched or sim), it now follows simulation rebalancing and exit rules **exclusively**. Subsequent manually-executed live adjustments are ignored by the sim to maintain strategy integrity.
*   **Divergent PnL Tracking**: Successfully decoupled the sub-strategy management from the `live_combined_portfolio`. The system now tracks both the "Simulation Path" and the "Broker Truth" as independent data streams on the dashboard.

## 🐛 Bug Fixes (Soft Bootstrap — Bugs 4–6)
*   **Bug 4 Fixed (tz-naive crash)**: `dt_series` stale reference replaced with `data['datetime'].iloc[0]` throughout chart-padding block. Bootstrap no longer crashes on mid-day deployments where DB data starts after 08:30.
*   **Bug 5 Fixed (time-gate on IC mapping)**: Replaced `ts.time() >= trade_start_time` guard with pre-assignment via 1-hour window matching. ICs filled just before a strategy's scheduled start are now correctly seeded.
*   **Bug 6 Fixed (fuzzy dedup)**: Removed `(minute, leg_sig)` deduplication block entirely. Order-ID deduplication is sufficient; the fuzzy check was incorrectly discarding valid ICs from strategies sharing the same strikes.

*   **`Trade` hashability fix**: `Trade` is a mutable `@dataclass` and not hashable. Using it directly as a `defaultdict` key in `trade_potential_strats` raised `TypeError: unhashable type: 'Trade'` in soft mode, silently killing the bootstrap. Fixed by keying on `id(lt)` instead.

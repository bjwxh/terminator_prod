## 1. Code Review & Performance Fixes (Implementations)
Based on the `CODE_REVIEW_2026-03-25.md` document, the following architectural fixes were applied to `server/core/monitor.py`:

- **Bug 5 & 8 Tracker**: Implemented a state-check to prevent duplicate "RECON" trades from being queued multiple times while an existing signal is still pending.
- **P6 Greek Cache Fix**: Removed redundant `timestamp` from the option lookup cache key. Since the cache is cleared per tick/snapshot, this increases performance by allowing multiple sub-strategies to share the same Greek lookup for identical strikes in the same interval.
- **Task #45**: Optimized the `order_queue` submission logic to ensure "atomic" registration of active order signals.
- **Cleanup**: Restored missing `_peak_equity` attribute required for calculating drawdown statistics.

## 3. Real-time VIX Integration (VIX Index)
A major feature integration was completed to display the VIX volatility index alongside the SPX price.

- **Discovery (`tmp/vix_test.py`)**:
    - Identified that Schwab streams the VIX index using the symbol `$VIX`.
    - Discovered that indices are streamed under the `LEVELONE_EQUITIES` service in `schwab-py` 1.5.1, rather than a separate index service.
- **Backend Integration (`server/core/monitor.py`)**:
    - Integrated `schwab.streaming.StreamClient` into the monitor's lifecycle.
    - Added a dedicated `_run_vix_streaming_loop` to handle WebSocket updates for `$VIX` in real-time.
    - Preserved REST API rate limits by relying entirely on WebSockets for index data.
- **WebSocket API (`server/api/ws.py`)**:
    - Updated the `state_update` broadcast to include the current `vix` value.
- **Web UI UI Changes**:
    - **`index.html`**: Added a new status entry for VIX in the header bar.
    - **`app.js`**: Updated the UI management logic to refresh the VIX display every 500ms when data arrives via the state broadcast.
